# Skill Mesh Balance

`$docara` owns Docara 2 site mechanics: project initialization, JSON settings,
navigation, locale wiring, static build/verification, publication handoff,
Docara-specific troubleshooting, and Docara/Larena product proof.

## Does Not Own

- Documentation method, information architecture, audience design, screenshot
  workflow, and content quality owned by `$docs`.
- SEO Contract and search acceptance owned by `$seo`.
- UX design of product/admin interfaces owned by `$ux`.
- QA evidence and readiness verdicts owned by `$tester`.
- Runtime/deploy/hosting incident mitigation owned by `$ops`.
- Platform facts outside Docara owned by `$bitrix`, `$sf4`, `$sf5`, `$larena`,
  or `$orgportal`.

## Companion Contracts

- Use `$docs` as author/reviewer/gatekeeper for substantial documentation
  content, docs maps, screenshots, and audience structure.
- Use `$seo` for public/indexable docs pages, canonical/title/H1/meta,
  structured data, sitemap, and robots expectations.
- Use `$tester` to verify links, build output, navigation, assets, and smoke.
- Use `$ops` for deployment/runtime/hosting issues.

## Handoff

Return Docara project paths, changed front matter/navigation/locales, build
commands, publication target, evidence, blockers, and what `$docs`, `$seo`, or
`$tester` must review next.
